# design_patterns
A collection of header-only C++ design patterns, each with its own dedicated file and usage examples.

---

## 📂 Project Structure
```text
📂 design_patterns
├── behavioral_pattern ................... # A behavioral pattern describes collaboration between objects.
│   └── observer ............................. # Define a one-to-many dependency between objects
|       |                                          where a state change in one object results in all its
|       |                                          dependents being notified and updated automatically.
│       ├── examples ......................... # Examples
│       ├── observer ......................... # Source Code
│       └── scripts .......................... # Scripts to compile and run
├── creational_pattern/ .................. # A creational pattern creates objects.
│   └── singleton ............................ # Ensure a class has only one instance, and provide a global point of access to it.
│       ├── examples ......................... # Examples
│       ├── scripts .......................... # Scripts to compile and run
│       └── singleton ........................ # Source Code
├── .gitignore ...................... # Git ignore file mapping untracked files
├── LICENSE ......................... # Project license file (GPLv3)
├── note.txt ........................ # Project local notes or internal checklist
└── README.md ....................... # Main project documentation file
```
